# کوتاه‌کننده لینک (Link Shortener)

## معرفی پروژه

<p>
این پروژه یک اپلیکیشن فرانت‌اند برای مدیریت و شبیه‌سازی فرایند کوتاه‌کردن لینک‌های طولانی است
که با React و JavaScript توسعه داده شده و برای استایل‌دهی از Tailwind CSS استفاده می‌کند.
</p>

<p>
هدف اصلی این پروژه، پیاده‌سازی یک رابط کاربری تمیز، قابل‌فهم و تعاملی برای سناریوی Link Shortening بوده است.
کاربر می‌تواند یک URL طولانی را وارد کند، برای آن زمان انقضا و Alias تعیین کند،
و در نهایت یک لینک کوتاه فرضی دریافت کند.
</p>

<p>
از آن‌جایی که این پروژه فقط در لایه فرانت‌اند پیاده‌سازی شده،
عملیات کوتاه‌سازی لینک به یک API واقعی متصل نیست و پاسخ‌ها به‌صورت شبیه‌سازی‌شده تولید می‌شوند.
</p>

## جزئیات پیاده‌سازی

<p>
در این پروژه تمرکز اصلی روی طراحی تجربه کاربری، مدیریت state، اعتبارسنجی ورودی‌ها،
مدیریت تاریخچه لینک‌ها، و پیاده‌سازی قابلیت‌هایی مثل Copy، Share، QR Code، Undo و Modalها بوده است.
</p>

<p>
همچنین با استفاده از Context API منطق برنامه از UI جدا شده
و برای نمایش پنجره‌های شناور از React Portal استفاده شده است.
</p>

## توسعه آینده

<p>
در صورت اتصال پروژه به بک‌اند، می‌توان قابلیت‌هایی مثل ذخیره‌سازی واقعی لینک‌ها در دیتابیس،
ریدایرکت سمت سرور، مدیریت واقعی تاریخ انقضا، ساخت Alias یکتا، احراز هویت کاربر،
و گزارش‌گیری از تعداد کلیک‌ها را نیز به آن اضافه کرد.
</p>

## ✅ موارد خواسته‌شده در تسک و وضعیت پیاده‌سازی

### 1) فرم ورودی URL

**خواسته تسک:**
- اینپوت برای URL
- دکمه ساخت لینک کوتاه
- نمایش خطا در صورت خالی بودن یا نامعتبر بودن ورودی

**پیاده‌سازی شده: ✅**
- فرم دریافت URL همراه با اعتبارسنجی:
  - بررسی خالی نبودن
  - بررسی فرمت صحیح URL (اعتبارسنجی دقیق‌تر)
- نمایش پیام خطا/موفقیت با Toast

**فایل مرتبط:**
- `UrlForm.jsx`

---

### 2) نمایش لینک کوتاه بعد از ارسال فرم (بدون API واقعی)

**خواسته تسک:**
- بعد از ارسال فرم، یک لینک کوتاه فرضی نمایش داده شود
- امکان شبیه‌سازی پاسخ API (نیازی به API واقعی نیست)
- نمایش پیام خطای واضح برای ورودی نامعتبر
- امکان کپی کردن لینک کوتاه

**پیاده‌سازی شده: ✅**
- شبیه‌سازی پاسخ API با `setTimeout`
- نمایش نتیجه در قالب مدال (Alert/Modal)
- قابلیت **Copy** و **Share**
- نمایش Toast برای موفقیت/شکست عملیات copy/share

**فایل مرتبط:**
- `LinkResultModal.jsx`

---

### 3) نمایش لیست لینک‌های ساخته‌شده در همان صفحه (History)

**خواسته تسک:**
- ذخیره لینک‌ها در `localStorage`
- نمایش دوباره لینک‌ها بعد از refresh
- امکان حذف لینک از لیست
- امکان سرچ داخل لیست

**پیاده‌سازی شده: ✅**
- ذخیره لینک‌ها در `localStorage`
- بازیابی و رندر مجدد بعد از refresh
- کارت‌ برای هر لینک
- حذف لینک
- جستجو در لیست لینک ها
- Undo بعد از حذف (Bonus)

**فایل‌های مرتبط:**
- `HistoryBox.jsx`
- `HistorySearch.jsx`
- `ShortLinkCard.jsx`

---

### 4) انتخاب زمان انقضای لینک (Expiry)

**خواسته تسک:**
- کاربر بتواند در فرم زمان انقضا را انتخاب کند

**پیاده‌سازی شده: ✅**
- انتخاب زمان انقضا با `select`
- نمایش برچسب/لیبل مربوط به expiry روی کارت‌ها

**فایل مرتبط:**
- `UrlForm.jsx`
- `ShortLinkCard.jsx`

## 📌 توضیح ساختار و رفتار کلی پروژه

- پروژه کاملاً کامپوننت‌محور است.
- بخش زیادی از Logic داخل Context قرار دارد و UI تا حد زیادی فقط مسئول نمایش است.
- مدال‌ها با Portal رندر می‌شوند تا کنترل بهتر روی لایه‌بندی و DOM وجود داشته باشد.
- لینک‌ها در `localStorage` ذخیره شده و بعد از refresh هم باقی می‌ماند.
- هر لینک در قالب یک کارت نمایش داده می‌شود که شامل:
  - لینک اصلی و لینک کوتاه‌شده
  - لیبل انقضا
  - دکمه‌های Copy / Share / Delete
  - نمایش QR Code و دانلود QR به صورت SVG
  - Undo بعد از حذف

## 🔮 اگر فرصت بیشتری داشتم، چه چیزهایی را بهتر می‌کردم؟

### ریزتر کردن کامپوننت‌ها
بعضی کامپوننت‌ها قابلیت تقسیم به بخش‌های کوچک‌تر را دارند تا خوانایی و reusability بهتر شود.

### خردتر کردن Logic و تبدیل به Custom Hook
با اینکه منطق از UI جداست و داخل Context قرار دارد، می‌توان بخش‌هایی مثل:
- copy/share
- validation
- localStorage
- shortenLink

را به hookهای مستقل تبدیل کرد تا تست‌پذیری و توسعه بهتر شود.

### استفاده از ابزار مدیریت state
در صورت بزرگ‌تر شدن پروژه می‌توان از ابزارهایی مثل `Zustand` یا `Redux Toolkit` استفاده کرد.

### توسعه به نسخه Full-stack
پیاده‌سازی پروژه با **Next.js** به صورت فول‌استک:
- API Routes
- دیتابیس (**MongoDB**)
- ذخیره‌سازی واقعی لینک‌ها

### اضافه کردن تست
اضافه کردن تست‌های ساده با **Vitest** و **React Testing Library** برای فرم، اعتبارسنجی و تاریخچه.
